free-gpu | provider workflow planner

Plan the right free or near-free compute path for the task in front of you.

llmfit answers what can run locally. free-gpu answers where the rest of the work should go: fast free notebooks, short-lived credit buckets, or grant-style providers when the task moves into heavier territory.

How the workflow planner thinks

The planner estimates the compute lane from workload, model size, target VRAM, job duration, and parallelism. It then schedules different provider types per stage instead of treating every task like the same generic ranking problem.

  • Burst lane for quick inference, demos, and short exploratory runs.
  • Session lane for longer notebook work, eval loops, and moderate fine-tunes.
  • Heavy lane for bigger VRAM asks or longer sustained runs.
  • Grant-scale lane for tasks that look more like allocations, credits, or formal programs than casual free-tier use.

What ships in the repo

  • CLI and Textual TUI for provider planning.
  • CSV-backed provider dataset and scoring heuristics.
  • MCP server so agents can ask for provider plans programmatically.
  • Role-aware ranking lenses for student, researcher, and founder contexts.

The planner keeps product semantics on purpose: role is a ranking lens, grants behave like a card-required lane, and budget buckets are UX buckets rather than literal accounting truth.

Compute lanes

Burst

Short jobs, minimal friction

Good for small inference tasks, quick demos, and experiments where setup speed matters more than raw headroom.

Session

Credits, notebooks, medium runs

Fits longer dev sessions, moderate eval workloads, and fine-tunes that still live in the near-free lane.

Grant-scale

Heavy tasks need a different lane

When VRAM, hours, or persistence grow, the planner shifts toward allocation and program-style providers instead of pretending casual free tiers will cover it.

Examples

CLI

free-gpu plan --workload finetune-lora ^
  --model llama-3.1-8b ^
  --budget under-25 ^
  --task-hours 6 ^
  --min-vram-gb 16

MCP

{
  "tool": "plan_provider_workflow",
  "arguments": {
    "workload": "agent-loop",
    "budget": "under-25",
    "task_hours": 3,
    "parallel_jobs": 4,
    "requires_api": true
  }
}

UI

free-gpu ui --task-hours 2 --min-vram-gb 12
